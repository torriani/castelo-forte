01 · Primeiro contato

Começar a usar (30 a 45 min)

O que você precisa instalar, clonar e configurar antes de tudo. Faça uma vez e nunca mais precisa.

Pré-requisitos

⚙️

Node.js 18+

brew install node no Mac. Confira com node --version.

🤖

Claude Code CLI

Baixe em claude.com/claude-code. Faça login.

🐙

GitHub CLI

brew install gh e depois gh auth login.

▲

Vercel CLI

npm install -g vercel e vercel login (necessário pra deployar o portal).

Passo a passo de instalação

  1. Clone o repositório

    cd ~/
mkdir -p claude && cd claude
git clone git@github.com:torriani/castelo-forte.git
cd castelo-forte

    Tamanho: ~740MB (inclui portal Vercel completo). Demora 2-5min na primeira vez.

  2. Instale dependências

    cd squads/conteudo/app/image-generator && npm install && cd -
cd squads/conteudo/skills/aprovacoes && npm install && cd -
cd apps/publisher-instagram && npm install && cd -

  3. Configure o .env (credenciais)

    cp .env.example .env
code .env  # ou: nano .env

    O administrador entrega os tokens pelo 1Password no item "Castelo Forte — Squad Conteúdo". Cole na ordem.

  4. Habilite o pre-commit hook

    git config core.hooksPath .githooks

    Esse hook bloqueia commits com arquivos >10MB e secrets. Protege a operação.

  5. Teste de fumaça

    node squads/conteudo/scripts/list-archive.mjs

    Deve listar pastas campanhas/, inteligencia/, multiplicar/ etc. Se der erro 401, peça token novo ao admin.

  6. Abra o Claude Code aqui

    cd ~/claude/castelo-forte
claude

    Dentro do Claude Code, ative o orquestrador:

    @content-chief

    Se ele responder ativando o Imperador do Squad, está tudo funcionando.

03 · Mão na massa

Como rodar na prática (4 fluxos)

Os 4 fluxos mais comuns do dia a dia, com comando exato, dependências locais necessárias e onde fica o resultado. Mostre essa seção pros operadores antes de eles começarem.

A) Subir conteúdo novo pro portal

Você criou carrosseis novos via squad e quer eles aparecendo no portal de aprovação. Aqui o fluxo completo:

  1. Copie os PNGs gerados pra pasta do portal

    Os carrosseis novos saíram em outputs/campanhas/{slug}/carrosseis/. Mova pro portal:

    cp outputs/campanhas/em-cristo/carrosseis/dia-22/*.png \
   apps/conteudo/img/cf-carrosseis/dia-22/

    Estrutura esperada: 1-capa.png + 2.png até 7.png (capa + 6 internos).

  2. Sincronize pro Supabase (banco do portal)

    node ~/claude/castelo-forte/squads/conteudo/skills/aprovacoes/scripts/publicar.mjs castelo-forte

    Escaneia apps/conteudo/img/cf-*, detecta novos itens, faz upsert no Supabase. Idempotente: pode rodar várias vezes sem duplicar.

  3. Commit + push no GitHub

    cd ~/claude/castelo-forte
git add apps/conteudo/img/cf-carrosseis/dia-22/
git commit -m "feat(portal): adiciona carrosseis do dia 22 da campanha em-cristo"
git push

  4. Deploy automático no Vercel

    Configuramos auto-deploy: push no main = deploy em ~30s. Não precisa rodar comando nenhum. Você vê o status em:

    vercel.com/torriani/castelo-forte/deployments

  5. Gere link mágico pro aprovador

    node ~/claude/castelo-forte/squads/conteudo/skills/aprovacoes/scripts/link.mjs castelo-forte

    Imprime URL única com token. Mande pro aprovador via WhatsApp.

⚡
Atalho: Em vez de lembrar tudo isso, rode /castelo-forte:conteudo:portal no Claude Code. Ele te guia passo a passo perguntando o que você quer fazer.

B) Criar uma campanha nova

Campanha = sequência integrada de peças (feed + stories + reels) em torno de um tema central, com cronograma dia-a-dia.

Comando único

cd ~/claude/castelo-forte
claude
@content-chief
*campanha

O que o Chief vai pedir (8 inputs)

InputObrigatório?Exemplo
Tema centralSim"A autoridade do filho de Deus no cotidiano"
Público-alvoSim"Membros 25-45 anos que se sentem derrotados"
ObjetivoSimAtração / Consciência / Aquecimento / Venda
DuraçãoSim3, 5 ou 7 dias
Oferta relacionadaNão"Encontro de Avivamento 25/05"
Formatos desejadosNão (default: carrossel + reels + stories)"só carrossel + stories"
Prints/provasNãoDepoimentos, screenshots, fotos
Crença a destruirNão"Que a vida cristã é só aguentar até morrer"

O que o Chief entrega

  • Narrativa central da campanha (fio condutor)
  • Progressão emocional ao longo dos dias
  • Cronograma dia-a-dia com formato + intenção de cada peça
  • Cada peça criada pelo agente especializado e validada pelo Anti-IA

Variante avançada (E1-E8)

Se a campanha é mais sofisticada (lançamento de evento, ofensiva editorial), use o @strategist direto com uma das 8 estratégias:

E1 — Lançamento de pressão (5 dias) E2 — Aquecimento suave E3 — Doutrina silenciosa E4 — Quebra de objeção E5 — Híbrido E6 — Reativação E7 — Manifesto E8 — Convocação evento

C) Multiplicar um YouTube (pregação/live/aula)

Pegar 1 pregação de 50min e gerar 30+ peças (carrosseis, reels, stories, frases) automaticamente.

Pré-requisitos locais (instalar 1 vez)

📦
Cada operador precisa ter: 
brew install yt-dlp ffmpeg
yt-dlp baixa áudio do YouTube, ffmpeg processa vídeos longos. Opcional (só se a transcrição nativa do YouTube falhar): OPENAI_API_KEY ou GEMINI_API_KEY no .env.

Comando único

cd ~/claude/castelo-forte
claude
@content-chief
*multiplicar

Você cola a URL

Chief pergunta: "Qual é a fonte do conteúdo pilar?"

  • 1. URL do YouTube ← cole aqui
  • 2. Colar texto/transcrição (se já tem)
  • 3. Arquivo local (áudio/vídeo)

Pipeline automático (7 fases)

1
Transcrever

Tenta transcrição nativa do YouTube primeiro (melhor qualidade). Se falhar, usa yt-text + Gemini. Último recurso: Whisper API.

↓
2
Limpar transcrição

Sonnet corrige pontuação, remove muletas verbais, formata em parágrafos.

↓
3
Extrair átomos

Cada átomo = 1 ideia isolada com potencial viral. Salva em outputs/multiplicar/{slug}/atomos/

↓
4
Planejar formatos

Chief decide quais átomos viram: carrosseis (3-5), reels (5-8), stories (2-3 sequências), frases (10-15), e-mails (0-3).

↓
5
VOCÊ APROVA O PLANO

Chief mostra o plano completo e PARA. Você revisa e aprova ou ajusta antes de criar as peças.

↓
6
Criar peças

Cada formato vai pro agente especializado: @carousel-creator, @reels-creator, @stories-strategist, @print-tweet-creator.

↓
7
Validar tudo (Anti-IA + Oráculo + tom)

@content-validator roda em cada peça. Reprovados voltam pra reescrita.

Onde fica o resultado

outputs/multiplicar/{nome-da-pregacao}/
├── transcricao.md
├── atomos/
│   ├── 01-autoridade-no-cotidiano.md
│   ├── 02-filho-nao-mendigo.md
│   └── ...
├── carrosseis/
├── reels/
├── stories/
├── frases/
└── emails/

D) Gerar imagens em lote (copy já pronta)

Você tem 20 copies prontas em texto e quer transformar em PNG. Há 2 caminhos dependendo do tipo de imagem:

Pré-requisitos locais (instalar 1 vez)

📦
Cada operador precisa: 
cd ~/claude/castelo-forte/squads/conteudo/app/image-generator
npm install
Instala Sharp (manipulação de imagem). Para gerar com IA também, precisa OPENROUTER_API_KEY no .env raiz.
🎨

Caminho 1 — Templates (carrosseis/print-tweets)

Se as imagens são slide de carrossel ou print-tweet (texto bem formatado sobre fundo). Usa template HTML + CSS, renderiza com Sharp. Rápido e consistente.

🤖

Caminho 2 — IA generativa (capas únicas)

Se a imagem precisa ser gerada do zero (capas ilustrativas, cenas únicas, conceitos visuais). Usa Gemini/Nano Banana via OpenRouter. Criativo mas custa por geração.

Caminho 1: Templates em lote

# 1. Crie o arquivo de lote (JSON com 1 item por slide)
cat > /tmp/lote.json <<'EOF'
{
  "defaultTemplate": "twitter-branco",
  "meta": { "author": "Castelo Forte", "handle": "@castelofortemandaqui" },
  "slides": [
    { "body": "Você não foi chamado pra **aguentar**.\nFoi chamado pra **reinar**." },
    { "body": "Filho não pede esmola pro próprio Pai." },
    { "body": "Sua autoridade não está no púlpito.\nEstá no nome." }
  ]
}
EOF

# 2. Rode o gerador apontando pra pasta de saída
cd ~/claude/castelo-forte/squads/conteudo/app/image-generator
node generate.mjs --batch /tmp/lote.json --output ../../../../outputs/frases/dia-15/

Templates disponíveis:

twitter-branco twitter-black imperial-dark claude-azul castelo-forte-editorial castelo-forte-frase

Catálogo completo: squads/conteudo/templates/catalog.yaml. Preview navegável: abra squads/conteudo/templates/CATALOGO.html no browser.

Caminho 2: IA generativa em lote

# 1. Crie o arquivo de lote (JSON com prompts)
cat > /tmp/lote-ia.json <<'EOF'
{
  "items": [
    {
      "prompt": "Cinematic photo of an open Bible on dark wood table, warm golden light from above, dust particles, ultra realistic, depth of field",
      "output": "outputs/capas/dia-15-biblia.png"
    },
    {
      "prompt": "Castle silhouette at dawn, cinematic, warm orange sky, christian church symbolism, ultra realistic",
      "output": "outputs/capas/dia-15-castelo.png"
    }
  ]
}
EOF

# 2. Rode o gerador IA
cd ~/claude/castelo-forte
bash squads/conteudo/scripts/gerar-imagem-ia.sh --batch /tmp/lote-ia.json --mode production
💰
Modo test vs production
--mode test usa modelo barato (riverflow-v2-fast) pra validar prompts antes de gastar. --mode production usa modelo final. Sempre teste em batch pequeno primeiro.

Atalho via squad

Em vez de montar o JSON na mão, rode no Claude Code:

@content-chief
*gerar-imagem-ia
# ou pra templates:
*render-slides

Os agentes montam o JSON pra você baseado nas copies que você cola e rodam o gerador.

04 · Operação diária

Portal de aprovação

Como acessar o portal, alterar conteúdos publicados, ver status da fila de aprovação e gerar links mágicos pro aprovador externo.

O que é o portal

Site estático na Vercel que serve 2 públicos:

  • Aprovador externo (pastor/liderança): recebe link mágico, aprova ou reprova peças com clique. Pode deixar observação.
  • Admin (você): gerencia clientes, vê estatísticas, dispara re-deploys.

URLs em produção

URLFunçãoAcesso
castelo-forte-ten.vercel.app/aprovar/?cliente=castelo-forte&token=<TOKEN>Aprovação externaToken mágico (gerado pela skill)
castelo-forte-ten.vercel.app/admin/AdminLogin: juliano@castelo-forte.com.br
castelo-forte-ten.vercel.app/conteudo/maio/Conteúdo publicávelPúblico

Como acessar pra editar (operador)

  1. Rode o portal localmente

    Importante: o portal é HTML/CSS/JS estático puro. Não tem npm run dev porque não é app Node/React. Use qualquer servidor estático:

    # Opção 1: Python (já vem no Mac)
cd apps/conteudo
python3 -m http.server 8080

# Opção 2: Node (npx, sem instalar nada)
cd apps/conteudo
npx serve -p 8080

# Opção 3: Vercel CLI (mais fiel à produção — respeita vercel.json)
cd apps/conteudo
vercel dev

    Abre em http://localhost:8080 (ou porta que o Vercel escolher). Cmd+C pra parar.

    💡
    Qual usar? Pra editar texto/imagens, qualquer um serve. Pra testar redirects, headers ou cache (config em vercel.json), use vercel dev — é o único que simula 100% o ambiente Vercel real.

  2. Edite os arquivos

    Pastas principais:

    • apps/conteudo/aprovar/ — interface do aprovador
    • apps/conteudo/admin/ — painel admin
    • apps/conteudo/conteudo/maio/ — material publicável do mês
    • apps/conteudo/img/cf-carrosseis/dia-NN/ — carrosseis por dia (capa + 6 internos)
    • apps/conteudo/img/cf-frases/dia-NN/ — frase por dia
    • apps/conteudo/assets/ — CSS, JS, fontes globais

  3. Deploy pra produção

    cd apps/conteudo
vercel --prod --yes

    ~30s e a mudança está no ar.

05 · Produção

Criar conteúdo novo

Fluxos passo a passo para cada tipo de peça. Todos passam pelo Content Chief (orquestrador) que diagnostica e delega.

Antes de criar (sempre)

cd ~/claude/castelo-forte
claude
@content-chief

O Chief carrega automaticamente:

  • Voice DNA da Castelo Forte (workspace/.../brand/voice/)
  • Regras inviolaveis (squads/conteudo/data/regras-inviolaveis.md)
  • Filtro Anti-IA (squads/conteudo/checklists/filtro-anti-ia.md)
  • Teologia do Reino como combustível da copy

Fluxo 1 — Carrossel

  1. Ative o Chief e peça briefing

    @content-chief
*briefing

  2. Responda 2 perguntas (no máximo)

    • "Qual o objetivo?" → atrair / mudar percepção / esquentar / vender
    • "Sobre o que?" → tema central

  3. Aprove a prescrição

    Chief recomenda: Tipo (Imperial/Polêmico/Crença/Problema/História/Oferta), Framework, Tamanho (1-10 slides), Abertura.

  4. Chief delega pro @carousel-creator

    Escreve, passa pelo @content-validator (Anti-IA + Oráculo + tom Castelo Forte), gera PNGs.

  5. Onde fica

    outputs/campanhas/{campanha}/carrosseis/{slug}/
├── post.md          # texto fonte
├── post.json        # dados estruturados
├── slide-01.png     # ... até slide-10.png
└── legenda.md       # caption do Instagram

Fluxo 2 — Reels

@content-chief
*reels

Pede: tema, duração (15s/30s/60s/90s), tipo (educativo/inspiracional/convocação/depoimento). O @reels-creator entrega roteiro com timestamps, texto na tela, B-roll, hook dos 2s iniciais, legenda.

Fluxo 3 — Stories

@content-chief
*stories

Pede objetivo + quantidade. O @stories-strategist entrega sequência (não stories soltos): gancho, desenvolvimento + interações (enquetes, caixinhas), CTA final.

Fluxo 4 — Print-tweet (reflexão)

@content-chief
*briefing  # diz que é print-tweet

Ou direto: @print-tweet-creator. Reflexão de 3-12 linhas em fundo branco estilo tweet/X. Template padrão: twitter-branco.

Fluxo 5 — Campanha (E1-E8)

@content-chief
*campanha

Pra lançamentos ou semanas temáticas. Chief delega pro @strategist que monta 1 das 8 estratégias:

E1 — Lançamento de pressão (5 dias) E2 — Aquecimento suave (10-14 dias) E3 — Doutrina silenciosa E4 — Quebra de objeção E5 — Lançamento híbrido E6 — Reativação de lista E7 — Manifesto/posicionamento E8 — Convocação para evento

Cada campanha distribui peças entre feed + stories + reels ao longo dos dias.

06 · Pipeline avançado

Multiplicar conteúdo longo

Quebrar 1 live, aula ou podcast em 30+ peças (carrosseis, reels, stories, frases, briefs de e-mail). O ativo mais subutilizado da igreja é a pregação gravada — esse fluxo extrai todos os átomos.

💡
Por que multiplicar? Uma live de 1h tem em média 20-30 ideias isoladas com potencial viral. Sem multiplicar, todas morrem na live. O *multiplicar garante que nenhum insight bom é desperdiçado.

Como rodar

  1. Ative o Chief e dispare o comando

    @content-chief
*multiplicar

  2. Informe a fonte

    Chief pergunta o que é:

    • URL do YouTube
    • Arquivo de áudio/vídeo local
    • Texto/transcrição já feita

  3. Deixe o pipeline rodar

    O pipeline tem 7 fases automatizadas:

    1. Transcrever — Whisper local pra áudio/vídeo
    2. Limpar transcrição — Sonnet corrige
    3. Extrair átomos — cada átomo = 1 ideia isolada com potencial viral
    4. Planejar formatos — Chief decide quais átomos viram quais formatos
    5. Pedir aprovação do plano ← você revisa
    6. Criar peças — cada formato com seu agente especializado
    7. Validar — Anti-IA + Oráculo em cada peça
    8. Distribuir no calendário — datas sugeridas

  4. Onde fica o resultado

    outputs/multiplicar/{nome-conteudo}/
├── transcricao.md
├── atomos/
│   └── {N}-{tema}.md
├── carrosseis/
├── reels/
├── stories/
├── frases/
├── tela-dividida/
└── emails/                  # briefs pra Squad Copywriters

Caso real: pregação de domingo

Uma pregação típica de 50min gera:

🎠

3-5 carrosseis

Ensinos centrais virados em sequências de 7-10 slides com ganchos próprios.

🎬

5-8 reels

Cortes potentes (15-60s) com hooks dos 2 primeiros segundos.

📱

2-3 sequências de stories

Convites pro próximo culto, enquetes de aplicação prática.

💬

10-15 frases

Quote cards de tweets, prints reflexivos.

07 · Distribuição

Publicar no Instagram

Pipeline automático com cron (LaunchAgent). Posts agendados são publicados nos horários definidos, com filtro Anti-IA bloqueante.

Adicionar post à fila

Cada post é uma pasta dentro de apps/publisher-instagram/publicar/ com nome no formato:

NN-YYYY-MM-DD-HHMM-slug/

Exemplo: 15-2026-05-25-1900-domingo-pentecostes/

Dentro da pasta, exatamente 3 arquivos:

ArquivoO que é
post.pngImagem do post (1080x1350 ou 1080x1080)
post.jsonMetadados (tipo, slug, agendado_para, campanha)
legenda.mdCaption do Instagram (até 2200 chars)

Comandos do dia a dia

ComandoO que faz
/publicar-statusStatus da fila (próximo agendado, falhas, etc)
/publicar-logLog do dia
/publicar-agora NForça publicação manual do post #N
/publicarPublicar post avulso (dry-run + aprovação)
/publicar-pauseDesativa cron
/publicar-resumeReativa cron

Gate Anti-IA bloqueante

🚫
Antes de publicar, o publisher roda: 
bash squads/conteudo/scripts/validate-anti-ia.sh legenda.md
bash squads/conteudo/scripts/validate-anti-ia.sh post.json --json
Se falhar: publicação BLOQUEADA. Você recebe notificação macOS + erro no log. Corrija a legenda e tente de novo.
08 · Compliance

Regras inegociáveis

TODO conteúdo gerado deve respeitar essas regras. Sem exceção.

Filtro Anti-IA (Layer 0, bloqueante)

Toda peça passa antes de publicar. Detecta travessão, sequências genéricas, jargão IA. Sem aprovação = não publica.

👑

Teologia do Reino

Combustível da copy. Filho não serviçal. Reinar agora, não só ir pro céu. Autoridade espiritual no cotidiano.

🎙️

Voice DNA da Castelo Forte

Carregado SEMPRE antes de criar. Arquétipos: Mago/Cuidador/Criador. Tom: forte mas sutil, profundo mas acessível.

❌

NUNCA usar travessão (—, –)

Substitua por vírgula, ponto, dois pontos, parênteses ou quebra de linha. Travessão denuncia IA escrevendo PT-BR.

🇧🇷

Português brasileiro completo

Acentuação obrigatória (á, é, ç, ã, õ, ê, í, ó, ú). Escrever sem acento é erro grave.

📁

Outputs no lugar certo

Sempre em outputs/, nunca dentro de squads/.

⚖️
O filtro Anti-IA é mecânico, não opinativo. Roda squads/conteudo/scripts/validate-anti-ia.sh que faz grep determinístico. Exit 0 = aprovado, exit 1 = corrigir. Compliance binário: 1 violação = reprovação total. Sem chance de "quase passou".

10 · Quando der errado

Problemas comuns

Top erros que aparecem na operação diária, com solução direta. Para problemas mais profundos: docs/TROUBLESHOOTING.md.

SintomaCausa provávelSolução
"ERRO: .env não encontrado" Não copiou template cp .env.example .env e edite
"Supabase list falhou (401)" Token errado ou expirado Peça token novo ao admin (1Password)
"@content-chief não responde" Você está fora da pasta do repo cd ~/claude/castelo-forte antes
"Publisher bloqueado" (notificação) Filtro Anti-IA reprovou Veja /publicar-log, corrija legenda, rode validate-anti-ia.sh
"Container error" no Instagram Imagem inválida (aspect ratio errado) PNG deve ser 1080x1350 ou 1080x1080
"Link mágico não funciona" Token expirou (30 dias) Gere novo: link.mjs castelo-forte
"Mudei portal mas não atualizou" Vercel ainda processando ou cache Aguarde 30s e dê Cmd+Shift+R
"pre-commit bloqueou meu commit" Arquivo >10MB ou .env staged Mídia vai pro Supabase, .env nunca commita
